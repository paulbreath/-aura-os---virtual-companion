# /review - 代码审查

自动检查代码质量、安全问题和最佳实践。

## 触发条件
- `/review` 命令
- "审查代码"、"检查代码"、"code review"

## 审查清单

### 1. TypeScript 类型安全
- [ ] 没有使用 `any` 类型
- [ ] 所有函数都有返回类型
- [ ] 接口定义完整
- [ ] 没有未使用的导入

### 2. React 最佳实践
- [ ] 组件使用函数式写法
- [ ] 正确使用 hooks
- [ ] 避免不必要的重渲染
- [ ] 使用 memo 优化性能

### 3. 安全检查
- [ ] 没有硬编码的 API 密钥
- [ ] 用户输入已验证
- [ ] 没有 XSS 漏洞
- [ ] 敏感数据不存 localStorage

### 4. 错误处理
- [ ] API 调用有错误处理
- [ ] 用户友好的错误提示
- [ ] 加载状态正确显示
- [ ] 没有未处理的 Promise

### 5. 代码质量
- [ ] 没有重复代码
- [ ] 函数职责单一
- [ ] 变量命名清晰
- [ ] 适当的注释

## 审查流程

```
1. 运行 npm run lint 检查类型错误
2. 检查改动的文件
3. 分析每个文件的问题
4. 生成审查报告
5. 提供修复建议
```

## 输出格式

```markdown
## 🔍 代码审查报告

### 通过项目 ✅
- TypeScript 类型安全
- React Hooks 使用

### 需要修复 ❌
1. **文件**: src/App.tsx:123
   - 问题: 使用了 any 类型
   - 建议: 改为具体的 Message 类型
   - 严重度: 中

2. **文件**: server.js:45
   - 问题: 缺少错误处理
   - 建议: 添加 try-catch
   - 严重度: 高

### 建议改进 💡
- 考虑使用 React.memo 优化列表渲染
- 提取重复的 API 调用到 service 层

### 修复命令
```bash
npm run lint  # 运行类型检查
```
```
